# 争端分歧机 Phase 2 - 以德服人版外包需求

## 一、背景
Phase 1是运气服人（剪刀石头布、骰子随机决定）。
Phase 2是以德服人（理由说服、打分投票），更严肃、更有说服力。

## 二、功能需求

### 2.1 说服型争端
场景：两个人有矛盾，各提交理由，由评委打分，分高者胜。
流程：
1. 创建者选择说服型争端，填写标题、描述
2. 设置正方反方，生成链接分享
3. 正方提交理由（文字）
4. 反方提交理由（文字）
5. 评委对每个理由打分（1-10分）
6. 统计总分，胜者获胜
功能：可设置匿名打分、可设置评委人数门槛

### 2.2 讨论型决策
场景：多人讨论一个问题，提交观点，匿名投票选最佳。
流程：
1. 创建者选择讨论型决策，填写问题
2. 每人提交观点（文字）
3. 匿名展示观点
4. 每人给每个观点打分
5. 计算平均分，最高分获胜

### 2.3 协议签署
场景：争端结束后生成协议确认页，增强仪式感。
功能：
- 显示争端结果和参与人列表
- 确认遵守结果按钮
- 生成协议截图（可分享）

### 2.4 匿名打分
功能：
- 创建时可选匿名模式
- 匿名模式下不显示打分者身份
- 显示每个选项平均分

## 三、技术要求

### 3.1 数据结构新增

reasons表（理由）：
- id: UUID
- dispute_id: 争端ID
- participant_id: 提交者ID
- content: 理由内容
- score: 分数
- is_anonymous: 是否匿名

scores表（评分）：
- id: UUID
- dispute_id: 争端ID
- reason_id: 被评分理由ID
- scorer_id: 评分者ID
- score: 分数(1-10)

### 3.2 接口新增
| 接口 | 方法 | 路径 |
|------|------|------|
| 提交理由 | POST | /api/disputes/:id/reasons |
| 评分 | POST | /api/disputes/:id/scores |
| 获取理由 | GET | /api/disputes/:id/reasons |

## 四、页面清单
| 页面 | 路由 |
|------|------|
| 以德服人-创建 | /debate/create |
