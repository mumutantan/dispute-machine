## 一、项目流程问题

### 问题1：跳过测试直接交付 ⚠️ 严重
- 现象：Phase 1 前后端开发完成后，直接报告完成，没有经过测试
- 影响：老板查看时发现链接打不开、功能不可用
- 教训：开发完成不等于交付完成，必须经过完整测试流程
- 改进：建立铁律 - 开发 → 测试 → 修复 → 再测试 → 交付

### 问题2：前端用了 mock 数据就交差
- 现象：像素写完了页面，但没有和后端 API 对接
- 影响：页面能看但不能用
- 教训：前端开发必须和后端 API 联调

### 问题3：没有给团队明确的技术约束
- 现象：齿轮用了 better-sqlite3 需要 C++ 编译，机器上没有 VS
- 教训：派任务前要了解环境限制

---

## 二、技术问题

### 问题1：better-sqlite3 需要 C++ 编译环境
- 解决：改用 sql.js（纯 JS 的 SQLite）

### 问题2：vant 包解压失败（Windows 路径过长）
- 解决：用 --ignore-scripts 先安装

### 问题3：中文文件写入被安全策略拦截
- 解决：先用 write 写 tmp，再用 Python 脚本 copy

---

## 三、工具与环境问题

### 问题1：没有 curl 和 gh CLI
- 解决：手动下载安装到 tools/ 目录

### 问题2：GitHub 推送 500 错误
- 原因：国内网络访问 GitHub 不稳定
- 解决：多试几次
