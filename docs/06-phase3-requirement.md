# 争端分歧机 Phase 3 - 账号与社交外包需求

## 一、背景
Phase 1-2是基础功能，用户无需登录即可使用。
Phase 3增加账号系统、社交功能，让用户能管理历史记录并分享结果。

## 二、功能需求

### 2.1 微信登录
流程：
1. 用户打开应用，弹出微信登录按钮
2. 点击后获取微信openid
3. 自动创建账号并登录
4. 显示用户头像和昵称

数据：
- users表：id、openid、nickname、avatar、created_at、last_login

注意：微信登录仅微信内置浏览器中可用，非微信环境可作为游客。

### 2.2 历史记录
功能：
1. 我的页面显示参与过的争端列表
2. 每个记录显示：标题、类型、结果、时间
3. 点击可查看详情

页面：
- 我的主页：/user/profile
- 历史记录：/user/history
- 争端详情：/history/:id

### 2.3 结果海报分享
功能：
1. 争端结束后生成结果海报
2. 海报包含：结果标题、获胜者、参与者、日期
3. 一键保存到相册或分享

### 2.4 分享裂变
功能：
1. 每创建一个争端生成专属海报
2. 海报带二维码
3. 朋友扫码即可参与

## 三、技术要求

### 3.1 微信登录实现
后端：
- 验证微信服务器请求的signature
- 通过code获取openid和session_key
- 生成JWT token返回前端

前端：
- 判断是否微信环境
- 微信环境显示登录按钮
- 非微信环境显示游客模式

### 3.2 海报生成
推荐前端使用html2canvas生成图片。

### 3.3 接口新增
| 接口 | 方法 | 路径 |
|------|------|------|
| 微信登录 | POST | /api/auth/wechat |
| 获取用户信息 | GET | /api/users/me |
| 获取历史记录 | GET | /api/users/history |
| 生成海报 | GET | /api/disputes/:id/poster |

## 四、页面清单
| 页面 | 路由 |
|------|------|
| 登录页 | /login |
| 我的主页 | /user |
| 历史记录 | /user/history |
| 分享海报 | /share/:id |

## 五、验收标准
- [ ] 微信环境中能正常登录
- [ ] 登录后能查看历史记录
- [ ] 争端结束后能生成结果海报
- [ ] 海报能保存到相册
- [ ] 非微信环境可作为游客使用
